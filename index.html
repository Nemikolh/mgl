MGL

mgl is a meta programming library for OpenGL. This library offers simpler way to perform OpenGL calls and manipulating OpenGL objects.

What can do this library for me ?

When using OpenGL core profile in C++ with at least the 3.0 features it offers, it isn't obvious to get a simple example to work. As soon as you're repeating yourself when writing the code, you feel like you should be wrapping some OpenGL calls. Thus you'll start wrapping some objects, starting to get a proper design working good with the OpenGL state machine. Then you end up writing a library. Not the basic example you wanted to code.

This library provides you a way to do this basic example without having to create the above library. It offers simple way to manage your buffers in GPU memory, fill them with data easily. And most importantly to bind automatically the underlying attributes for the shader stage.

If the library does some of the job in a simpler way, it still requires from you a good knowledge of how OpenGL works.

Basic example

A basic example is available under the folder examples. It shows how to use the wrapper mgl::gl_vector<T> type, the MGL_DEFINE_GL_ATTRIBUTES macro, the vao creation with mgl::gl_program::make_vao(T...) member function and the mgl::gl_draw function which are the base of the library.

Note that the automatic binding of attributes work pretty well, however the automatic generation of the shader isn't properly implemented yet. It does not what we would like it to do. :)

Install

The library is aimed to be header-only. Currently in order to get your own version of mgl working, you will need the following additionals librairies:

  • GLEW
  • glm
  • Boost.Fusion
  • Boost.MPL
  • Boost.Preprocessor
  • Boost.TypeTraits

Note that the librairy isn't in alpha yet. Several features will likely be changed over the next months. The dependencies on GLEW and Boost will probably be removed over time. However, the dependency on glm won't change as mgl has been thought to be well integrated with glm.
